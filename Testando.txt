git init -> cria a pasta .git
git status -> verifica o status antes do commit
git add "[nome arquivo].[tipo]" -> add um arquivo pra commitar
git reset -> remove um arquivo que iria ser commitado (remove do add)
git add . -> add todos arquivos para commitar
git commit -m "[mensagem]" -> commita com a mensagem
git remote add origin https://github.com/ArthurZL/[nome do repositório] -> vincula o repositório criado no git ao push que será dado
git push --set-upstream origin master -> cria a branch master e sobe os arquivos para ela
git reset HEAD~1 - Deu commit, mas não deu push e quer voltar
git revert HEAD -> deu commit e pushed e quer voltar, reverte os commits locais
git reflog -> monstra o histórico
git reset --har [id fornecido no reflog] -> aponta para a versão escolhida
git pull -> Traz as informações mais atualizadas do repositório remoto
git branch -> monstras as ramificações disponíveis
git branch [nome qualquer] -> cria um novo ramo com o nome definido
git checkout [nome da branch] -> muda da branch atual para a branch que foi passado o nome
git push --set-upstream origin [nome da branch a enviar] -> subir para o GitHub essa nova branch já com as alterações
git merge [nome da branch que será puxada] -> Uni na branch que está atualmente o conteúdo da branch que tem o nome que foi inserido no comando
git checkout -b [nome da branch que será criada] [nome da branch a se basear] -> cia nova branch usando de base a branch inserida
